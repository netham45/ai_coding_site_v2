# Stack Decision Record

## Purpose

Provide one explicit place for the repository to record its chosen languages, frameworks, persistence tools, test tools, UI stack, and any still-open stack questions.

The starter repo ships this as a blank contract on purpose.

The starter should not decide these choices ahead of the cloned project's own genesis and architecture work.

### Open Questions

- Which stack choices are still unresolved?
- Which decisions are blocking setup work?
- Which decisions can remain deferred safely?

### Decision Notes

- Why were these choices made?
- What alternatives were rejected?
- What constraints or risks influenced the decision?

## Rule

Once setup begins to depend on concrete stack choices, this file and the related inventory, verification-command, and architecture notes should be updated in the same change.
