Overlay Role Delta
- Use with `plan/base.md`.
- This overlay turns the plan into an execution plan for one implementation slice.
- Keep proof and repository updates attached to the task that creates the change.

Overlay Objective Delta
- Produce crisp execution tasks that leave implementation reviewable and bounded-proof-ready.

Additional Forbidden Actions
- Do not create one giant "implement the whole plan" task.
- Do not split only by file count.
- Do not push required proof or updates into an unspecified later task.
- Do not satisfy the plan by doing task-owned implementation yourself.

Profile-Specific Expected Result
- A task child set for implementation work.

Profile-Specific Completion Conditions
- Each task has a crisp execution boundary.
- Proof and repository updates remain attached to the task that creates the change.
- The task set is small enough to execute cleanly but large enough to close real work.
