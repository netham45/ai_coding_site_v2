# Original Starting Idea

## Purpose

Preserve the user's rough concept before it gets normalized into a more structured repository note.

## Raw Idea

Create an option for something like "collaborative design" where:

- the AI creates the pages you want with a basic design
- the AI then asks the user to review the design
- the AI asks for a list of design requirements the page should have
- those requirements may include concrete UI fields, labels, sections, and other specific design constraints
- the workflow should help an operator who has little or no formal UI-design experience
- external tools might be relevant, such as Figma or something else, but the idea is still rough

## Immediate Interpretation

The strongest part of the idea is not "generate a pretty page."

The strongest part is:

- generate an initial concrete draft
- force a review checkpoint
- translate vague human feedback into structured design requirements
- revise against those requirements instead of relying on one-shot prompting

That is closer to a workflow contract than to a single prompt tweak.

## Placement Hint

This appears to be a post-`workflow_overhaul` concept because it depends on richer workflow-profile behavior rather than only better prompt wording.
