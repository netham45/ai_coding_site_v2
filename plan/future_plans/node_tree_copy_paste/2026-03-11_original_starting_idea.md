# Original Starting Idea

Random future plan:

Make a system so I can copy whole node trees.

It should copy from the top-level node to the bottom-level subtask.

It should paste stopped, with a fresh snapshot of the base repo, with the project selected during paste and that snapshot getting applied at the epic level.

From there the parent would start all of its children with their original prompts.

Those children would do their thing and then start their children, and so on until all of the tasks have run.

Basically what would be skipped is the actual child-generation stage.

Epics, phases, and plans would instead be told:

"You were created with predefined children. Here they are."

This should be considered after the web UI and workflow-overhaul feature plans.

Along with this plan, there should also be a way to export a node tree so it can be passed around and shared.
