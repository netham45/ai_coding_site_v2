# Task Plans

This folder contains task-scoped plans that govern concrete implementation or doc-alignment work.

Task plans are authoritative documents.

Schema references:

- `notes/catalogs/checklists/authoritative_document_family_inventory.md`
- `notes/catalogs/checklists/document_schema_rulebook.md`
- `notes/catalogs/checklists/document_schema_test_policy.md`

Task-plan rules:

- every meaningful code change must be associated with a task plan under `plan/tasks/`
- task plans use the repository standard richer plan schema shared with feature plans
- task plans must also include `Verification` and `Exit Criteria`
- task plans must define the canonical verification commands to run before the task can be described as verified
- task plans must name the notes and adjacent feature plans that define the task boundary
