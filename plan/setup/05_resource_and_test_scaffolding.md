# Phase S05: YAML, Prompt, And Test Scaffolding

## Goal

Establish the non-code resource skeleton and the testing structure required for the rest of implementation.

## Scope

- Database: prepare fixture helpers for resource-linked DB tests.
- CLI: prepare test harnesses for CLI-resource integration.
- Daemon: prepare daemon-resource integration test helpers.
- YAML: create built-in system-yaml directory families and placeholder files.
- Prompts: create prompt-pack directory families and placeholder assets.
- Tests: create exhaustive fixture layers for YAML loading, prompt loading, compiler setup, runtime simulation, and performance baselines.
- Performance: add reusable benchmark helpers for compile cost, resource load cost, and command latency.
- Notes: keep `notes/yaml_builtins_checklist.md` and prompt notes aligned with the scaffold.

## Exit Criteria

- resource tree exists.
- test harness can load placeholder YAML and prompt assets.
